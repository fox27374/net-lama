Passively senses nearby networks and client stations in
      monitor mode. The agent adapts the scan: first run sweeps all channels,
      then narrows to "interesting" channels where activity was heard, cutting
      scan time from ~50s to ~15s on busy phys. Needs a monitor-capable adapter
      and a privileged (root) agent.

    

    

      
Actively connects to the SSID: association, authentication,
      DHCP and optional throughput — each step timed. The sensor's radio leaves
      monitor mode for the test and returns afterwards (min. interval 300s).

      

        
        
      

      

        
        
      

      

        
        
      

      
      
      
⚠ Randomized MAC uses a fresh address every run — each test consumes a DHCP lease (can exhaust the pool) and shows up as a new device in the AP's client table. Use Permanent for a fixed sensor.

    

    

      
Measures throughput and latency between two agents. The test runs on the
      source agent only, and is auto-assigned to the source agent's site. The destination must have
      its perfmon reflector enabled with a reachable advertise host (Agents page → Edit) —
      agents without one aren't listed below.
